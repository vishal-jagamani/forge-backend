# Backforge Codebase Index

## Repo Shape
- `src/`: CLI runtime, prompts, generator pipeline, constants, shared types, and utilities
- `template/`: scaffold templates copied into generated projects
- `dist/`: built CLI output generated by `tsup`
- Root docs/config: `README.md`, `CHANGELOG.md`, `package.json`, `tsconfig.json`, `tsup.config.ts`, lint/format config

## Runtime Entry
- `src/cli.ts`: CLI bootstrap, startup output, prompt execution, generator execution, success messaging, and centralized error handling

## CLI Flow
1. `src/cli.ts` starts the CLI and prints the startup banner.
2. `runBasicPrompt()` in `src/prompts/index.ts` asks for a simple test input and logs it.
3. `runPromptFlow()` in `src/prompts/prompt-flow.ts` collects project setup input.
4. The generator pipeline in `src/generator/index.ts` creates the target folder, resolves the template, copies files, replaces placeholders, and optionally installs dependencies.
5. `src/cli.ts` prints the generated project location and next-step commands.

## Prompts (`src/prompts`)

### Prompt Orchestration
- `src/prompts/prompt-flow.ts`: builds the final `CliConfig` from CLI args and interactive prompt responses
- `src/prompts/index.ts`: exports prompt modules and still contains a basic input prompt used during current CLI startup/testing

### Prompt Modules
- `src/prompts/project-name.prompt.ts`: asks for the project name, validates it, and normalizes the result
- `src/prompts/config.prompt.ts`: collects supported language, architecture, and package manager selections
- `src/prompts/preferences.prompt.ts`: collects whether dependencies should be installed after generation

## Generator (`src/generator`)
- `src/generator/index.ts`: main generation orchestration
  - resolves target directory
  - blocks generation if the folder already exists
  - resolves the correct template path
  - copies template files
  - replaces placeholders such as `{{projectName}}`
  - optionally installs dependencies
- `src/generator/resolve-template.ts`: maps CLI config to `template/<language>/<architecture>` and validates that the template exists
- `src/generator/copy-template.ts`: recursively copies template files while skipping `node_modules`
- `src/generator/replace-placeholders.ts`: walks generated files and replaces template placeholders in supported text formats
- `src/generator/install-deps.ts`: runs `npm install`, `pnpm install`, or `yarn install` in the generated project directory and falls back gracefully on failure

## Constants (`src/constants`)
- `src/constants/prompt.constants.ts`: prompt labels, defaults, and user-facing choice strings
- `src/constants/generator.constants.ts`: ignored directories/files and allowed text extensions used during placeholder replacement
- `src/constants/index.ts`: barrel export for constants

## Types (`src/types`)
- `src/types/prompt.types.ts`: enums and interfaces for language, architecture, package manager, and setup preferences
- `src/types/cli-config.ts`: final configuration shape returned by the prompt flow and consumed by the generator
- `src/types/index.ts`: barrel export for shared types

## Utilities (`src/utils`)
- `src/utils/project-name.utils.ts`: npm-safe project-name validation plus kebab-case normalization
- `src/utils/index.ts`: barrel export for utility helpers

## Templates (`template`)

### Template Conventions
- `template/README.md`: explains the directory convention `template/<language>/<architecture>/`, placeholder usage, and static-file-only expectations

### Current Implemented Template
- `template/ts/functional/`: current v1.0.0 starter template
  - `package.json`: generated project metadata and scripts
  - `tsconfig.json`: TypeScript config for the generated project
  - `eslint.config.mjs`: lint setup for generated apps
  - `.prettierrc`, `.prettierignore`, `.gitignore`: formatting and ignore defaults
  - `.env`, `.env.example`: starter environment files
  - `src/app.ts`: Express app setup
  - `src/server.ts`: server startup
  - `src/config/env.ts`: environment loading with `dotenv`
  - `src/routes/index.ts`: health route and sample route mounting
  - `src/modules/sample/*`: sample controller, service, and routes

### Placeholder/Future Template Directories
- `template/ts/oop/`
- `template/js/functional/`
- `template/js/oop/`

## Build and Tooling
- `package.json`: package metadata, executable `bin` mapping, scripts, dependencies, and versioning
- `tsconfig.json`: compiler config and internal path aliases
- `tsup.config.ts`: CLI bundling configuration targeting `dist/cli.js`
- `eslint.config.mjs`: ESLint setup for the Backforge codebase
- `.prettierrc`: formatting rules
- `.prettierignore`: formatting exclusions
- `.gitignore`: ignored dependencies, build output, env files, logs, and editor/system artifacts

## Current Supported Surface in v1.0.0
- Language: `TypeScript`
- Architecture: `Functional`
- Package managers exposed in prompts: `pnpm`, `npm`
- Dependency installation: optional
- Generated app type: Express backend starter

## Output Surface
- CLI command: `backforge`
- Local dev entry: `pnpm dev`
- Build command: `pnpm build`
- Current runtime result: interactive scaffold generation into a new project directory

## Important Notes
- The codebase still contains `runBasicPrompt()` in `src/prompts/index.ts`, which appears to be temporary/testing behavior in addition to the main prompt flow.
- `package.json` currently contains a local `link:` dependency for `backforge`, which looks development-specific rather than publish-ready.
- `template/ts/oop`, `template/js/functional`, and `template/js/oop` are placeholders and are not implemented in `v1.0.0`.
- `CHANGELOG.md` should stay user-facing; keep deeper implementation detail here in `CODEBASE_INDEX.md`.
