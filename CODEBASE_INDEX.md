# Backforge Codebase Index

## Repo Shape
- `src/`: CLI source code, prompts, shared types, and utilities
- `dist/`: built CLI output generated by `tsup`
- Root config files: package metadata, TypeScript config, linter, formatter, and bundler setup

## Runtime Entry
- `src/cli.ts`: CLI bootstrap, startup logs, prompt execution, final config output, and centralized error handling

## Prompt System (`src/prompts`)

### Prompt Orchestration
- `src/prompts/prompt-flow.ts`: runs the full prompt sequence, merges CLI args with interactive answers, and returns the final `CliConfig`
- `src/prompts/index.ts`: exports prompt modules and includes a basic input prompt used during current CLI testing

### Prompt Modules
- `src/prompts/project-name.prompt.ts`: collects the project name, validates it, and returns a normalized value
- `src/prompts/config.prompt.ts`: collects language, architecture style, and package manager selections
- `src/prompts/preferences.prompt.ts`: collects setup preferences such as dependency installation and Git initialization

## Types (`src/types`)
- `src/types/prompt.types.ts`: enums and interfaces for project language, architecture, package manager, and setup preferences
- `src/types/cli-config.ts`: top-level CLI config shape returned by the prompt flow
- `src/types/index.ts`: barrel export for shared prompt-related types

## Utilities (`src/utils`)
- `src/utils/project-name.utils.ts`: validates npm-safe project names and formats user input into kebab-case
- `src/utils/index.ts`: barrel export for utility helpers

## Build and Tooling
- `package.json`: package metadata, CLI bin mapping, scripts, and dependency definitions
- `tsconfig.json`: TypeScript compiler settings and path aliases for `@prompts/*`, `@app-types/*`, and `@utils/*`
- `tsup.config.ts`: bundler configuration for compiling the CLI into `dist/cli.js`
- `eslint.config.mjs`: ESLint setup using the base JS and TypeScript recommended configs
- `.prettierrc`: formatting rules used across the codebase
- `.prettierignore`: files and folders excluded from formatting
- `.gitignore`: ignored dependencies, build output, environment files, logs, and editor/system artifacts

## Current CLI Flow
1. The CLI starts in `src/cli.ts` and prints startup output.
2. `runBasicPrompt()` asks for a simple input value and logs the result.
3. `runPromptFlow()` reads an optional CLI arg for the default project name.
4. The project name prompt validates and formats the selected name.
5. The config prompt collects language, architecture, and package manager.
6. The preferences prompt collects install and Git init choices.
7. The combined config object is printed for the next generator phase.

## API and Output Surface
- CLI command: `backforge`
- Dev entry: `pnpm dev`
- Build entry: `pnpm build`
- Current runtime output: prompt-driven config collection printed to stdout

## Important Notes
- The generator/scaffolding stage is not implemented yet; the CLI currently collects and prints configuration only.
- `src/prompts/index.ts` still includes a basic name prompt used for testing alongside the full prompt flow.
- The package currently includes a local `link:` dependency entry for `backforge`, which may be intended for local development rather than publishing.
- Release history is documented in `CHANGELOG.md`; keep that file user-facing and reserve implementation details for this index.
